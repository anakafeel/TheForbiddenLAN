This page covers running the stack on your local machine for development and testing.

## Prerequisites

| Tool | Version | Notes |
|------|---------|-------|
| Node.js | 20+ | LTS |
| Docker | 24+ | Real Docker via get.docker.com — not snap |
| pnpm | 10.23.0 | Managed via Corepack |
| Android Studio | Latest | For mobile emulator testing |

---

## Setup

<Steps>

<Step>

### Clone and Bootstrap

```bash
git clone <repo-url>
cd TheForbiddenLAN
pnpm setup:local
```

`pnpm setup:local` enables Corepack, pins pnpm to 10.23.0, installs all workspace dependencies, and runs Android environment checks.

</Step>

<Step>

### Start the Server

Create `packages/server/.env`:

```bash
POSTGRES_USER=skytalk
POSTGRES_PASSWORD=skytalkpass
POSTGRES_DB=skytalk
JWT_SECRET=dev-jwt-secret-change-in-prod
DATABASE_URL=postgresql://skytalk:skytalkpass@localhost:5432/skytalk
```

Then start Docker Compose:

```bash
cd packages/server
docker compose up -d
```

Docker Compose starts Postgres 16 and the Node.js server. On first start, the server runs `prisma db push` to create all tables, then listens on port 3000.

Verify it's up:

```bash
curl http://localhost:3000/ping
# → {"pong":true}
```

</Step>

<Step>

### Seed Development Data

```bash
docker compose exec server npx tsx prisma/seed.ts
```

Creates:
- `admin` / `admin` — role: admin
- `pilot1` / `test` — role: user
- Talkgroup: **Ground Ops** (both users as members)

</Step>

<Step>

### Mobile App

The app connects to the relay server on startup. Make sure the server is running (or use the hosted instance at `134.122.32.45:3000`).

```bash
cd packages/mobile
pnpm dev:mobile      # browser mode — fastest iteration
# or
pnpm android         # Android emulator
```

Configure the server URL in `packages/mobile/.env`:

```bash
EXPO_PUBLIC_API_URL=http://134.122.32.45:3000
EXPO_PUBLIC_WS_URL=ws://134.122.32.45:3000/ws
EXPO_PUBLIC_DLS140_URL=http://192.168.111.1:3000
EXPO_PUBLIC_LOOPBACK=false
```

</Step>

<Step>

### Admin Portal

```bash
cd packages/portal
pnpm dev
```

Opens at `http://localhost:5173`. Requires an admin JWT in `localStorage`. Get one:

```bash
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin"}'
```

Then in the browser console:

```js
localStorage.setItem('jwt', '<paste-jwt-here>')
```

</Step>

</Steps>

## Current Feature Status

| Feature | Status |
|---------|--------|
| PTT audio record + relay + playback | ✓ Working (Opus 16kHz CBR, two-device tested) |
| AES-GCM-256 encryption | ✓ Working (hardcoded test key) |
| Floor control (walk-on prevention) | ✓ Server-authoritative |
| Talkgroup membership | ✓ From server REST API |
| GPS signal | DLS-140 integration exists, untested on hardware |
| Presence | ✓ From server WebSocket |
| Loopback testing | ✓ Single-device (EXPO_PUBLIC_LOOPBACK=true) |

<Callout type="warn">
The crypto polyfill is a pass-through shim. Audio is sent with AES-GCM but the key is hardcoded. Production requires proper KDF and key rotation.
</Callout>
