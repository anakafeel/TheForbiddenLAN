SkyTalk is a push-to-talk satellite communication system for aircrew operating in remote areas. It must feel like a walkie-talkie while running over **Iridium Certus** — a 22 kbps uplink, 500–1500 ms latency, expensive-per-byte satellite link.

Every decision documented here exists to answer one question: **how do you make that feel like a walkie-talkie given those constraints?**

---

## The Constraint That Drives Everything

The SKYTRAC DLS-140 satellite router on each aircraft is outbound-only. The Iridium network uses carrier-grade NAT — no inbound connections are possible. This makes direct aircraft-to-aircraft communication physically impossible. Every phone connects outbound to a relay server on DigitalOcean, and the server fans audio out.

This is not a design choice. It is a hardware constraint we designed around.
